Permissions to install sabre/katana

It seems sabre/katana does not have the correct permissions to be installed.

Don't worry.

How to fix?

You have basically two options. Most of the time, all you have to do is either to change the “mode” of the files or to change the “owner” of the files.

Changing the modes

You want your files to be readable and writable by your HTTP server and sabre/katana. Only files in the data/ directory are impacted. Thus, in a terminal, run:

$ cd path/to/katana/
$ chmod -R a+rw data/

Changing the owners

Another solution is your want your files to have the same owner that your HTTP server (or PHP FastCGI) instance. Again, only files in the data/ directory are impacted. Thus, in a terminal, run:

$ cd path/to/katana/
$ chown -R www-data:www-data data/

The user and the group are commonly www-data. It could also be www or _www.
